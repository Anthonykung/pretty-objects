Pretty Objects ✨

Make giant object payloads actually readable.

A tiny formatter and inspection tools for large structured object payloads. Pretty Objects is designed to make large or messy structured data human readable and easy to edit. It is developed by AI engineer to work on large AI datasets without going insane.

Why I made this

For real dataset work, not just tiny demo JSON.

Anyone working on AI stuff knows the pain of giant JSON and JSONL files that are basically impossible to read or edit without going insane. Pretty Objects is designed to make those files actually readable and manageable, with features that helps you tame the chaos without mangling your data.

JSON + JSONL Better readability for dataset and log workflows.
Safe repairs Conservative by default so your data does not get mangled.
Folding support Collapse the chaos without breaking valid structure.

Shortcuts

Keybindings

By default, the normal format keybinding is left alone so you can choose to use Pretty Objects there or not by setting it as the default formatter for JSON / JSONL. But there are also dedicated shortcuts for calling Pretty Objects commands directly, which you can customize in Keyboard Shortcuts.

Shift Alt F

Use this for JSON / JSONL when Pretty Objects is the selected default formatter.

Ctrl Alt Z

Cmd Alt Z (macOS)

Dedicated Pretty Objects shortcut when you want to call it directly.

Heads up

A few important notes

  • JS / TS and Python support can work on full programs too, but only by conservatively rewriting supported embedded literals.
  • Parse failures do not rewrite your content unless a bounded repair succeeds.
  • Collapse / expand uses editor folding, so the underlying JSON / JSONL stays valid.
  • Object Viewer uses virtual temporary editors, so saving there updates the source document without creating normal files.
  • Best-effort text-to-JSON fallback only runs when repair mode is set to bestEffort.
  • To get rid of the temporary editor tabs, you can simply reload the window with Developer: Reload Window and they shall never be seen again.
See it in action

Run guided demos

Each demo opens sample data in a before/after diff view so you can inspect the result immediately. The Object Viewer demo also opens the live viewer alongside its companion diff. Demo tabs are temporary and auto-close when you run another demo, close Welcome, or reload the window.

1

Prettify + repair preview

Opens malformed JSON and launches the preview diff so you can see conservative repair plus formatting, including trailing commas and obvious missing-comma cleanup.

Run Prettify Demo

2

JSONL pretty view

Opens sample NDJSON in a diff, showing the line-oriented input beside the readable review view used for dataset inspection.

Run JSONL Demo

3

Object Viewer workflow

Opens a companion diff plus the live Object Viewer so you can try configurable page sizes, filters, bulk actions, warnings, per-item diff, and save-back behavior.

Run Object Viewer Demo

4

Collapse nested objects

Opens a diff for the nested payload before and after formatting, then lets you apply the folding workflow yourself on the formatted view.

Run Folding Demo

5

Best-effort text to JSON

Opens plain text plus the converted JSON result so you can inspect the fallback used by Object Viewer saves in bestEffort repair mode, including : and = fields, bullet lists, repeated keys, and indentation-based nesting.

Run Text To JSON Demo

6

TypeScript literal payloads

Opens a real TypeScript source file and launches a preview diff for the source file so you can see Pretty Objects rewrite supported embedded literals inside normal program code too.

Run TypeScript Demo

7

Python literal payloads

Opens a real Python source file and launches a preview diff so you can see supported dict and list literals get cleaned up inside normal program code.

Run Python Demo
